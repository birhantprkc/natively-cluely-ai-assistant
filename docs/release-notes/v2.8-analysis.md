# Natively v2.8 Release Analysis Report

This report presents an exhaustive, evidence-based audit of the Natively v2.8 release cycle, comparing the current `HEAD` against the v2.7 baseline (`be7280b`). All claims are backed by commit history and code-level verification.

---

## Executive Summary

The Natively v2.8 release represents a major evolution of the desktop assistant's intelligence, performance, and operational footprint. The core of this release is the **"Intelligence OS" (IOS) architecture**—a 20-phase, highly structured refactor consolidated under a new `electron/intelligence/` layer. 

Designed for modularity, safety, and backward compatibility, the entire IOS layer is **flag-gated and default-OFF**. Under normal operations (flags disabled), the production code remains 100% path-identical to the v2.7 baseline, preserving all existing benchmarks and ensuring zero regressions.

### Key Release Highlights
1. **Intelligence OS (IOS):** 19 new modules under `electron/intelligence/` introducing deterministic profile voicer gating, context fusion, structured meeting extraction, layered conversation memory, and lecture/diagram service engines.
2. **Hindsight Long-Term Memory:** Optional integration with `@vectorize-io/hindsight-client` for cross-meeting recall, isolated by user/org bank scopes and strict tag filters.
3. **Regional STT Relay Migration:** Realtime audio transcription is migrated from Railway to region-local VPS instances (US and Singapore), cutting costs (~$25–$40/mo flat vs. ~$150+/mo Railway linear egress) and lowering latency.
4. **Smart Gemini Cascade:** Fallback ladder (`gemini-3.1-flash-lite` → `gemini-3.5-flash` → `gemini-3.1-pro-preview`) with shared keys, replacing the old tail-latency parallel racing mechanism.
5. **Overlay UI Polish:** Floating overlay resizing from 600px to 780px using Framer Motion springs, with hover-gated mouse passthrough for transparent margins.

## 1. New Features Added

### 1.1 "Intelligence OS" Core Infrastructure
Introduces a comprehensive suite of helper services to orchestrate context, memory, and routing. These are added strictly under the `electron/intelligence/` directory:
* **`ProfileTreeService`**: Deterministically resolves candidate resume facts and routes identity questions to avoid vector store lookups.
* **`LiveTranscriptBrain`**: Manages sliding windows, rolling summaries, and entity extraction over live, in-progress meeting segments.
* **`ContextRouter`**: Centralizes query-routing decisions (e.g. detecting if a query requires diagram generation or lecture memory).
* **`ContextFusionEngine`**: Merges up to 14 distinct data inputs, resolving conflicts by priority and capping total tokens.
* **`PromptAssemblerV2`**: Translates fused context into structured, trust-tagged XML blocks (<profile_tree>, <live_transcript>, etc.).
* **`MeetingMemoryService`**: Extracts entities, questions, decisions, and action items from transcripts.
* **`SearchOrchestrator`**: Fuses vector, lexical, and metadata search with configured weights.
* **`ConversationMemoryService`**: Tracks session turn history, resolving follow-up questions.
* **`LectureIntelligenceService`**: Generates structured revision materials, checklist outlines, and flashcards.
* **`DiagramIntelligenceService`**: Generates, validates, and labels Mermaid diagram markup.

### 1.2 Hindsight Long-Term Memory Adapter
Adds first-class long-term memory support via `@vectorize-io/hindsight-client` (Commit `3485d8d`, `a8acbb4`). 
* Retains meeting summaries asynchronously via a bounded backpressure queue.
* Recalls past context for backward-looking queries, bounded by strict `Promise.race` timeouts (100ms) to ensure LLM answer paths never block.
* Fully optional; falls back to `NoopMemoryProvider` when disabled.

### 1.3 Regional STT Relay
Realtime Speech-to-Text (STT) is migrated from the main Railway application server to regional VPS servers (Singapore & US) using the custom `@natively/stt-relay-core` library (Commit `0dbb05e`, `05ddc5e`).
* Verifies short-TTL HMAC session tokens offline at the relay.
* Supports fallback routing from regional servers back to the control plane.
* Metering usage is flushed every 45 seconds to Supabase.

### 1.4 Browser Extension Companion (`natively-browser/`)
Introduces a companion extension package (Commit `aa19663`, `natively-browser/`) to extract tab DOM context.
* Uses Mozilla Readability or custom InnerText fallbacks to extract clean text.
* Captures verbatim code blocks from active Monaco or CodeMirror editors (up to 25k chars limit).
* Detects user selection to prioritize highlighted text as the main answer context.

## 2. Existing Features Improved

### 2.1 Eager-Expand Overlay Resizing
The manual overlay panel width expansion is improved from a rigid layout shift to a smooth spring animation (Commit `9baec89`, `6d091ec`).
* Smoothly transitions width from 600px (collapsed) to 780px (expanded) centered in a fixed-width window, avoiding transparent window re-rasterization flickers.
* Gutter-pinned floating resize button (`ResizeToggle.tsx`) rides the panel's top-right corner.

### 2.2 Mode Reference Indexing
* **Pre-indexing Reference Files:** Files uploaded to modes are indexed, chunked, and embedded at upload time (Commit `6047c8d`, `6130ef6`) rather than delaying live queries.
* **Pre-warming Prompt Cache:** Warmed automatically on mode activation (Commit `6047c8d`) to ensure sub-millisecond retrieval.

### 2.3 LiteLLM Gateway Routing
* Integrates rate-limiting, key rotation, and automated fallback capabilities via `litellm.Router` configuration (Commit `7dfaa48`).

## 3. Bugs Fixed

### 3.1 Durable Memory Window Eviction
* **Issue:** The `getContext(7200)` call in `IntelligenceEngine` was meant to provide a 2-hour live memory window. However, `SessionTracker.evictOldEntries` evicted transcript items after 120 seconds, silently capping live memory.
* **Fix:** Introduced `SessionTracker.getDurableContext()` (Commit `84f1177`, `SessionTracker.ts:366`), which bypasses the short-term eviction list to query the durable `fullTranscript` store directly, restoring the true 2-hour window.

### 3.2 Floating Resize Button Misalignment
* **Issue:** The overlay resize toggle button misaligned off the diagonal when expanded due to asymmetric horizontal/vertical offset clamps.
* **Fix:** Rewrote offset logic in `NativelyInterface.tsx` to calculate a single diagonal offset `d` shared by both axes, pinning the button exactly on the 45-degree bisector.

### 3.3 Phone Chat false-supersession
* **Issue:** Phone chat streams were sometimes false-superseded by concurrent desktop chat streams, leading to broken mobile connections.
* **Fix:** Introduced `_phoneChatLatestId` in `ipcHandlers.ts` (Commit `8417ecb`) to isolate phone streaming supersession from desktop session identifiers.

### 3.4 Stale done-event token clobbering
* **Issue:** Slow, superseded streams could emit delayed `gemini-stream-done` events, overwriting newer, active streams.
* **Fix:** Passed `streamId` with all `gemini-stream-done` and `gemini-stream-token` events (Commit `8417ecb`), allowing the renderer to ignore outdated stream endings.

### 3.5 STT Relay missing Token `sub_id`
* **Issue:** Security review identified that signed STT relay tokens lacked the account uuid (`sub_id`), which would cause Supabase CHECK constraint failures on usage flushes.
* **Fix:** Added the `sub_id` to token generation and verified it via the unit suite before final staging (Commit `0dbb05e`, `abaffb6`).

## 4. Performance Improvements

* **In-Memory Transcript Processing:** All sliding-window, summary, and context-retrieval methods inside `LiveTranscriptBrain` execute fully in memory. Verified latency is **< 0.2ms**, far below the 250ms target.
* **Async Memory Buffering:** Hindsight memory writes use `HindsightRetainQueue` (Commit `a8acbb4`), which buffers and flushes summaries out of the main thread path.
* **Pre-flight STT caching:** Client auth requests are cached within the token's Time-To-Live (TTL), preventing redundant REST calls on connection drops.

## 5. Architecture Changes

```
            ┌───────────────── electron/intelligence/ (consult-only layer) ─────────────────┐
  query →   │ ContextRouter ─► {useProfileTree, useLiveTranscript, useHybridRag,            │
            │                    useHindsightRecall, useMeetingSummary, useLectureMemory,    │
            │                    useDiagramIntelligence, answerContract, maxLatencyMs}        │
            │      │                                                                          │
            │  ProfileTreeService   LiveTranscriptBrain   SearchOrchestrator  MemoryManager   │
            │  Lecture/Diagram       ConversationMemory    LongTermMemory(Noop|Hindsight)     │
            │      │                                                                          │
            │  ContextFusionEngine ─► PromptContextContract ─► PromptAssemblerV2 ─► XML+report │
            │  IntelligenceTrace / IntelligenceMetrics (observe-only)                         │
            └────────────────────────────────────────────────────────────────────────────────┘
    EXISTING live answer paths (UNCHANGED, flags OFF): ipcHandlers → IntelligenceEngine → WhatToAnswerLLM
```

* **Decoupled STT Data Plane:** Replaces the monolithic control plane proxying with regional VPS instances. Egress bandwidth moves to flat-rate VPS plans (~$25–$40/mo vs. ~$150+/mo Railway linear egress).
* **Consolidated Intelligence Layer:** All AI context logic is grouped under `electron/intelligence/`, establishing a clean, typed interface (`PromptContextContract`) for prompt synthesis.

## 6. UI/UX Changes

### 6.1 "Intelligence" Settings Panel
Adds a new panel (Commit `6047c8d`, `src/components/settings/IntelligenceSettings.tsx`) exposing:
* Toggles for all 16 feature flags (organized under Memory, Search, Answer Quality, Lecture & Diagrams, and Advanced groups).
* Config fields for Hindsight Cloud/Local URL and API key with a live health status indicator.
* "Try it" preview buttons for Lecture Notes, Diagrams, and In-Meeting Search.

### 6.2 Spring-Animated Resize
* The floating resize pill expands and collapses the sidebar width. Under the hood, Framer Motion springs drive the layout dynamically.

### 6.3 Deep-Linked System Permissions
* Audio-capture and screen-recording warnings now deep-link directly to the matching macOS System Preference Pane (`system` or `mic`) instead of launching a generic URL scheme that breaks on Windows.

## 7. Telemetry & Observability Changes

* **`IntelligenceMetrics`**: Timers track p50 and p95 metrics for live window lookups, global search, and context routing.
* **`piTelemetry`**: Emits new structured markers:
  - `pi_scaffold_compressed`: Triggered when structured output is condensed for speakability.
  - `pi_answer_repeated`: Fired when repetition guard catches duplicated content.
  - `pi_context_policy_applied`: Emitted when fallback rules are triggered.
* **STT VPS Observability**: Regional relays expose Fastify `/metrics` (active sessions, bytes, errors) and log structured `[STTEvent]` entries to AXIOM/Sentry.

---

## 8. STT / LLM / Provider Changes

### 8.1 Gemini Cascade Fallback
* Replaces parallel hedging/racing with a strict serial fallback ladder: `gemini-3.1-flash-lite` → `gemini-3.5-flash` → `gemini-3.1-pro-preview`.
* Honors the user's selected Gemini model as the starting rung. If Pro is selected, it executes only Pro. Falls forward only on pre-commit errors.

### 8.2 Gemini Flash-Lite Integration
* `gemini-3.1-flash-lite` is added as a first-class model in `VisionProviderRegistry.ts` (Commit `5021a2a`), serving as a fast, cost-efficient default for OCR and vision-first tasks.

---

## 9. Profile Intelligence & Context Routing

* **Candidate Perspective Verification:** `ProfileTreeService` deterministically checks if a query risks leaking assistant identity or "Natively" branding.
* **Neutralizing Prompt Injections:** `ContextFusionEngine.ts` inspects incoming blocks and strips out instructions matching high-risk patterns (e.g. "ignore previous directions").
* **Low-Trust Budgets:** Injections or low-trust sources are budgeted and truncated from the bottom, while verified system prompts and user profile contexts are preserved.

---

## 10. Security, Privacy, & Auth Changes

* **HMAC Signed STT Access:** Railway signs short-lived (180s TTL) HMAC tokens. VPS relays verify this offline to prevent unauthorized stream hijackings.
* **Participant Anonymization:** Hindsight adapters hash participant IDs before indexing, ensuring raw user metadata is never stored.
* **Search Isolation:** `SearchOrchestrator` applies strict per-user and per-org filters to search queries before ranking, preventing cross-tenant leakage.

---

## 11. Backend, Server, & API Changes

* **New STT Routes:** `natively-api` adds `POST /v1/stt/session` (session minting), `GET /v1/stt/relays` (health registry), and `/admin/stt-relays/control` (kill-switch).
* **Durable Billing Migration (`003`):** Replaces close-only billing with a state machine checking in every 45s. Heartbeat bypasses and double-billing are resolved structurally at the database constraint level.

---

## 12. Electron, Main-Process, & IPC Changes

* **`set-overlay-interactive-region`**: Allows the renderer to dynamically tell Electron whether the pointer is over the visible panel or the transparent margin, gating window click-through policies.
* **Settings IPCs:** Wires `intelligence-flags:get`, `intelligence-flags:set`, `hindsight-config:get`, `hindsight-config:set`, and `hindsight-config:test`.
* **Action IPCs:** Wires `lecture:generate-notes`, `diagram:generate`, and `search:in-meeting`.

## Rollout & Rollback Guidelines

All new services are deployed behind feature flags, defaulting to **OFF**. 

### Recommended Rollout Sequence
1. Enable `trace` to monitor background metrics.
2. Flip `profileTreeV2` to secure candidate profile questions.
3. Turn on `answerDiversityGuard` and `durableMemoryWindow`.
4. Turn on `meetingMemoryV2` and `inMeetingSearchV2`.
5. Roll out `lectureIntelligenceV2` and `diagramIntelligence`.
6. Enable Hindsight config: set the `baseUrl`, check health status, and toggle `hindsightMemory`.

### Rollback Protocol
If any regression is detected:
* **Feature Flags:** Set the environment variable or Settings tab toggle to `off`. Toggles take effect instantly on the next query.
* **Emergency Rollback:** To completely revert, delete the `electron/intelligence/` folder and revert the single additive edit in `electron/SessionTracker.ts`. The legacy v2.7 answer pipeline is fully intact.
